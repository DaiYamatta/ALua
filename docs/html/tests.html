Cenário Descrição
1 Um Daemon
1.1 Criar de daemon
1.2 Criar um daemon e conectar o processo roteador a ele.
1.3 Enviar um comando do processo roteador para o daemon executar.
1.4 Criar um novo processo Lua associado ao daemon e realizar troca de mensagens entre o novo processo e o daemon.
1.5 Criar dois novos processos Lua associados ao daemon e realizar troca de mensagens entre eles.
1.6 Criar novo processo Lua no daemon. Processo roteador troca mensagens com o processo Lua no daemon.
1.7 Criar dois processos Lua, um no daemon e outro no processo roteador. Realizar troca de mensagens entre eles.
2 Rede de daemons
2.1 Criar dois daemons e conectá-los. Realizar troca de mensagens entre eles.
2.2 Criar três daemons e conectá-los. Realizar troca de mensagens entre eles.
2.3 Criar três daemons e conectá-los. Cria mais um daemon e conectá-lo somente a um dos três anteriormente ligados em rede. Realizar troca de mensagens entre eles.
BUG Quando um daemon se conecta a uma rede de daemon já estabelecida, é possível enviar mensagem dele para qualquer outro daemon já pertencente a rede. Porém o inverso não é possível.
2.4 Em uma rede de três daemons criar em cada daemon da rede um processo Lua. Realizar troca de mensagens entres os processos.
2.5 Criar três daemons. Conectar apenas dois em rede. Enviar mensagem de um daemon da rede para o daemon fora da rede. Não será possível enviar esta mensagem.

3.3 Criar um novo identificador único no formato textual.
4 Rawsend
4.1 Registra um par (nome, socket) para enviar dados (função setfd)
4.2 Envia dado para um nome registrado (função send). Verificar se o retorno da função é igual a 1 e se os dados foram recebidos.
4.3 Envia dado para um nome não registrado (função send). Verificar se o retorno da função é igual a -1.
4.4 Envia dado para um nome registrado (função send) de múltiplas fontes em paralelo. Verificar se o retorno da função é igual a 1 e se os dados foram recebidos.
5 CCR
5.1 Verificar se o valor de ismain é true
5.2 Criar um novo processo (função spawn)
5.3 Buscar no processo principal pelo novo processo. Não será encontrado pois não foi registrado.
5.4 Registrar o novo processo
5.5 Buscar no processo principal pelo processo registrado. Será encontrado.
5.6 Remover o processo.
5.7 Buscar no processo principal pelo novo processo. Não será encontrado pois não está mais registrado.
5.8 Criar dois processos e registrá-los.
5.9 Processo principal envia dados para os dois novos processos.
5.10 Processos "filhos" verificam se existe dado a receber (função hasdata). Deve retornar true.
5.11 Processos "filhos" recebem os dados (função receive)
5.12 Processos "filhos" verificam se existe novo dado a receber (função hasdata). Deve retornar false.
5.13 Processos chamam a função receive e aguardam o recebimento de dados. Processo principal envia os dados.
5.14 Processos filhos chamam a função tryreceive para verificar se há dados, como não há continuam a execução
